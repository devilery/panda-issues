abinoda commented Nov 9, 2018

@majormoses Based on earlier comments here, the issue I was trying to tackle is that previously there was no way to exclude things like "WIP" from personal reminders at all. In the past, when users brought this up, I replied by saying that they should simply not assign reviews until the pull request was out of a WIP state. However, for teams that use codeowners, review requests are automatically generated when the pull request is opened, even if the pull request is in a WIP state. This made it very inconvenient to abide by my suggestion of not assigning or un-assigning reviewers until a review is actually needed. Therefore I decided to add filtering capabilities so that WIP, etc. can be excluded from personal reminders. See @ashanbrown's comments earlier as well.

I'm not understanding what you're looking for, but I think it should probably be requested as a separate issue?
